    Hosting

    Define the problem

    • Web site performance suffers due to out-dated infrastructure software

      • Current platform is running on AWS EC2 Ubuntu 12.04.5 LTS with PHP 5.3.
      • Ubuntu 12.04.5 end of life support on April 28, 2017. Upgrade asap is critial.
      • PHP 5.3 end of life support on Aug 14 2014.
      • Drupal 7 minimum recommended PHP5.4. Drupal 8 recommended PHP 7.2.

    • Web site is not able to handle HTTPS traffics

      • Some content has hardcode http links.
      • Infrastructure set up is not able to direct HTTPS traffic correctly.
      • LDAP or oAuth authentication is not possible

    • Complex infrastructure setup with redirect, load balancers, vanish

      • Difficult to debug when problem occurs.
      • True client IP can not detected resulting users got blocked from login.

    • Server recently become very fragile and slow

      • Deploying code could bring server down when cache clear
      • Logged in editors suffer longer delay when editing content for unknown reason

    • Deployment requires many coordination

      • Webteam is not able to deploy to staging and live environment and has to rely on other person who has greater priviledge, which makes deployment very inflexible.

    • Isolation for testing new feature is cumbersome

    • Limited access and total control

      • When something goes wrong, webteam may not be able to pingpoint the problem quickly due to limit access and deploy quick fix to the problem.

    • Lack of information on how the entire infrastructure is set up

      • No one within the webteam oversea the entire system.
      • No one within the webteam understand the infrastructure and integration.

    • Lack of Drupal expert support at the infrastructure level

    • Disaster recovery (rollback) strategy is unclear

    Identify the criteria

    • Performance - how the site performce under heavy load
    • Scalability - can the site scale up/down to meet demand
    • Stability - how the site can stay up without service interruption
    • Technical support at speedy fashion - how easy to reach expert support
    • Multi dev support - how easy to spin up a new test environment
    • Ease of deployment - how easy to deploy new change to production
    • Ease of data migration - moving data from one envioment to another
    • Customization - deployment hook, seperate configuration per env etc
    • Log analysis - any utility for monitor site traffics and logs
    • Intergration (oAth, Coldfusion, splunk etc)
    • User management - How easy to set up account to allow users have different level of access (External, Internal)
    • Backup and Disaster recovery - how easy/quickly to recovery
    • Cost - How much it will cost to run the stack
    • Server attack mitigation - how to avoid/mitigate server attack such as Ddos, brute force login attempt etc.
    • Data protection and privacy - how data is store and treat, who can access the data
    • Storage capacity (See https://pantheon.io/docs/platform-considerations/#highly-populated-directories)
    • Development tooling - Utility to assist develpment/management
    • Complexity - how easy to use the platform
    • Peace of mind/stress - how much support the platform provider can provide
    • Github/CI integration
    • Limitation on modules use
    • Standard Drupal distribution (no alteration of core)
    • Other CMS support
    • Custom upstream - provide a custom bootstrap standard site
    • Technical support resource
    • Email service - SMTP server
    • Media Streaming
    • Dev site protection (password) - how easy to set up basic protection for a dev site
    • Dealing with large file delivery (> 50mb)
    • Static IP - ability to bind a site to a static IP
    • Nginx/Apache, customization for server config
    • Migrating the current site to another platform
    • Environment consistency - configure environment with the same set up
    • Git based source control
    • Security on platform - how well the platform protect against intrusion

    Ease of deployment (K8: 2, Pantheon: 5, Acquia: tbd)

    K8 (2)

    • Since there no intuitive UI interface, deployment most likely will utilize commandline interface. Less tech savvy users may found it difficult to use or have a steeper learning curve.
    • Permission set up for users may also less straight foward.
    • If something goes wrong, rollback could also be complicated.
    • It is not sure at this point the webteam will have fully access for deployment, which is a higly desirable feature.

    Pantheon (5)

    • It provides a simple use UI that allow site admin to deploy from one environment to another.
    • With EDU plan, user role with various permission could be set up via the UI.
    • Backup can be schecule nightly/manually, if something mess up, rollback could be easily done.
